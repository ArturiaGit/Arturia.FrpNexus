# 真实 Linux VPS 闭环验收操作计划

## Summary

用一台测试 Linux VPS、一个本地 HTTP 服务和 FrpNexus 开发版，验证 v1.0 最小闭环：节点保存、SSH 连接、FRP 准备、TOML 生成、SFTP 上传、远程启动、状态刷新、日志读取、远程访问和数据持久化。预览发布、INI、快捷键/无障碍都暂不测试。

## 准备工作

- 准备一台测试 VPS，不要用生产机器。
- VPS 建议 Ubuntu/Debian，确认你能用普通 SSH 工具登录。
- 云厂商安全组和 VPS 防火墙放行：
  - SSH 端口，例如 `22`
  - FRP 测试访问端口，例如 `18080`
- 本机启动一个简单 HTTP 服务：
  ```powershell
  cd <任意测试目录>
  python -m http.server 8080
  ```
- 优先使用私钥路径或 SSH Agent 登录；如果用密码，只作为 session 输入，不保存。
- 在项目根目录先跑：
  ```powershell
  dotnet build
  dotnet test D:\Arturia\E\PersonalProjects\Arturia.FrpNexus\Arturia.FrpNexus.sln
  ```
- 本轮不要运行 `scripts/publish-win-x64.ps1`。

## 验收步骤

1. 节点页：
   - 新建 VPS 节点。
   - 填 Host、SSH 端口、用户名、认证方式、远程 FRP 目录等普通字段。
   - 保存后重启 FrpNexus，确认节点仍在。
   - 点击 SSH 测试连接。
   - 通过标准：节点可持久化，SSH 成功；失败时有中文错误，不泄露密码或私钥内容。

2. 隧道页：
   - 新建一个 HTTP 隧道。
   - 本地地址填 `127.0.0.1`。
   - 本地端口填 `8080`。
   - 远程端口填 `18080` 或你放行的端口。
   - 关联刚才的 VPS 节点。
   - 保存后重启应用，确认隧道仍在。
   - 通过标准：隧道可保存、可读取、可编辑。

3. 配置页：
   - 基于节点和隧道生成 TOML。
   - 点击验证语法。
   - 保存配置版本。
   - 上传 TOML 到 VPS。
   - 通过标准：TOML 内容符合隧道参数，验证通过，配置可保存和上传。

4. FRP 准备：
   - 下载或选择 FRP release。
   - 选择 Linux x64 对应的 `frpc` 或 `frps`。
   - 上传到 VPS 远程目录。
   - 通过标准：下载、选择、上传成功；错误信息可读且不含敏感值。

5. 远程运行：
   - 启动远程 FRP 进程。
   - 刷新状态。
   - 停止，再重启。
   - 再刷新状态。
   - 通过标准：启动/停止/重启/刷新都有明确状态；运行记录和部署记录正常更新。

6. 日志页：
   - 读取远程 FRP 日志。
   - 刷新日志。
   - 查看 WARN/ERROR 行是否可读。
   - 通过标准：日志能读取；失败时中文提示；日志不写入 SQLite。

7. 真实访问：
   - 在浏览器或命令行访问：
     ```text
     http://<VPS_IP>:18080/
     ```
   - 通过标准：能看到本机 `python -m http.server 8080` 暴露的目录页面。
   - 停止远程 FRP 后访问应失败；重启后访问恢复。

8. 安全与持久化：
   - 重启 FrpNexus。
   - 确认设置、节点、隧道、配置版本、运行记录、部署记录仍在。
   - 检查导出数据、SQLite、日志中不包含：
     - SSH 密码
     - Token
     - 私钥内容
     - 私钥 passphrase
   - 私钥路径或显示名可以存在，前提是它不是秘密本身。

## 失败记录模板

每发现一个问题，按这个格式记：

```text
页面：
操作：
期望结果：
实际结果：
错误文案：
是否可重试：
是否阻断闭环：
是否涉及敏感信息：
截图或日志片段：
备注：
```

## 通过标准

这次闭环验收通过，至少要满足：

- 一个真实 VPS SSH 测试成功。
- 一个 HTTP 或 TCP 隧道能完整配置并远程访问成功。
- FRP 二进制和 TOML 能上传。
- 远程启动、停止、重启、状态刷新和日志读取可用。
- 应用重启后普通数据仍在。
- 敏感数据没有被持久化、记录日志或导出。
- `dotnet build` 和完整 `dotnet test` 通过。

## Assumptions

- 先只测 HTTP 隧道，TCP/UDP/HTTPS 可以作为后续扩展回归。
- 本轮不发布预览包。
- 本轮不测 INI。
- 本轮不测快捷键和无障碍。
