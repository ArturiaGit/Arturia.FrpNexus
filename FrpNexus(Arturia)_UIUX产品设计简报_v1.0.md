# 【PRD】FrpNexus (Arturia)_产品需求与视觉简报_v1.0

> **"The promised tunnel to victory. (穿透壁垒的誓约之剑)"**
> **文档版本**：v1.0
> **产品代号**：Project Avalon (Arturia)

## 一、 产品概述 (Product Overview)

### 1. 核心痛点
官方 FRP 虽然底层网络穿透能力极强，但在非专业用户桌面端和多平台服务器运维中存在三大痛点：
* **配置门槛高**：依赖手写 TOML 配置文件，极易出错。
* **缺乏可视化**：无法直观监控连接状态、流量与日志。
* **守护进程繁琐**：在 Linux/无头服务器下配置开机自启和后台守护门槛较高。

### 2. 产品定位
一款基于 .NET 8/9 构建，秉承 **Arturia (极简与守护)** 理念的 FRP 跨平台智能管理终端。
**核心理念**：将复杂网络配置降维为填表操作，以“一处代码，双模运行”兼顾桌面 GUI 与无头服务器 CLI 场景；同时赋予专业工具独特的极客美学与灵魂。

### 3. 目标用户群体
* 需要暴露本地环境进行调试的研发工程师（对 UI 审美和极客文化有要求）。
* 个人独立开发者 / 极客玩家（轻量级、跨平台需求）。
* 中小企业 IT 运维（重度 CLI 依赖，需要快速批量部署节点）。

## 二、 品牌精神与命名规范 (Brand & Philosophy)

产品代号 **Arturia** 来源于《Fate》系列中的骑士王（阿尔托莉雅·潘达拉贡）。FRP 的核心功能与此意象高度契合：**守护本地服务的安全（阿瓦隆），同时像利剑一样穿透防火墙的网络壁垒（誓约胜利之剑）。**

### 1. 核心设计策略：抽象致敬 (Abstract Homage)
为保持专业工具的高端质感，全局采用“只取其神，不取其形”的策略。绝不使用任何动漫原图，而是将骑士王的元素抽象为**色彩、动效与命名系统**。

### 2. 技术架构与代码命名 (Tech Stack & Architecture)
* **全局命名空间**：`Arturia.FrpNexus`
* **交互与展示层**：`Avalonia UI` + `CommunityToolkit.MVVM` (GUI) / `Cocona` (CLI)
* **核心业务逻辑层（彩蛋命名）**：
  * `AvalonDaemon`（进程守护模块）：接管 FRP 核心生命周期与异常重启，意为绝对防御的剑鞘。
  * `ExcaliburTunnel`（穿透连接模块）：负责编译 TOML 并建立内网穿透连接，意为刺穿壁垒的圣剑。
  * `InvisibleAirService`（隐蔽服务模块）：负责 Linux 下的静默服务注册与桌面端托盘隐藏。

## 三、 核心功能演进路线 (Product Roadmap)

### 阶段一：MVP（最小可行性产品）- "跑通闭环，确立调性"
* [x] **双模运行 (Dual-Mode)**：双击拉起高颜值的 Avalonia 桌面端；带参数运行则为适配 Linux/macOS 的纯终端守护进程。
* [x] **工作台 Dashboard**：支持多套穿透方案的增删改查与运行状态监控。
* [x] **配置双向绑定**：UI 表单与 FRP 标准 TOML 文件的自动互转。
* [x] **Arturia 专属终端**：CLI 模式下炫酷的 ASCII Art 字符画与名台词启动语。

### 阶段二：增强版 (v1.5) - "专业好用，建立壁垒"
* [ ] **一键服务注册**：CLI 自动在 Linux 生成 `systemd` 开机自启脚本。
* [ ] **全自动环境准备**：自动探测 OS 架构拉取对应 FRP 核心程序。
* [ ] **静默结界**：关闭主窗口后，最小化到系统托盘持续提供穿透服务。

### 阶段三：完全体 (v2.0) - "数据化与云生态"
* [ ] **可视化流量看板**：调用 FRP Admin API，实时绘制出入站流量图表。
* [ ] **配置云同步**：支持 WebDAV 或 GitHub Gist 配置备份。

## 四、 用户体验链路演示 (UX Journey)

### 场景 A：独立开发者在桌面端的操作 (GUI)
1. 打开 `FrpNexus`，极简的深蓝色启动图闪过 `Crafted by Arturia`。
2. 进入工作台，点击新建穿透方案，通过卡片式表单填入本地 IP 和端口。
3. 点击金黄色的“启动”按钮（鼠标 Hover 时闪过一道剑气光泽动效）。
4. 状态指示灯从灰色平滑过渡为**耀眼金（Excalibur Gold）**，下方终端面板输出：`[AvalonDaemon] Tunnel established successfully.`

## 🎨 附录：UI/UX 设计简报 (Design Brief)

> **To 设计师**：
> 本产品视觉需融合**“微软 Fluent 现代质感”**与**“Fate 骑士王高定配色”**。我们需要的是高雅、极简、极具科技感的专业工具质感。

### 1. 界面框架规范：基于 WinUI 3 深度定制
本产品的交互和控件视觉需严格遵循微软 **WinUI 3 (流畅设计体系)**，并将我们的专属配色作为系统级变量注入：
* **材质与背景 (Mica / Acrylic)**：在支持的系统上开启背景模糊。底层映射色使用主品牌色（深蓝），结合半透明亚克力材质，打造深邃且通透的“防御阵”质感。
* **圆角规范**：严格遵循 WinUI 3，浮窗、卡片、按钮统一采用 `4px` 或 `8px` 圆角，体现现代桌面的温润感。
* **卡片式布局**：配置列表使用 WinUI 3 标准的 `Expander`（可折叠面板）或卡片式列表管理，具备清晰的层级投影（Elevation & Drop Shadow）。
* **图标规范**：全局使用微软开源的 **Fluent Icons**（线条圆润的线性图标）。

### 2. 核心色彩体系 (Color Palette)
全局采用**深色模式（Dark Mode）**，使用以下专属主题色覆盖 WinUI 3 默认配色：
* **主品牌色 (Saber Deep Blue)**：`#0F1E3D` (深群青/夜空蓝)。用于全局背景色、侧边栏底色。
* **核心强调色 (Excalibur Gold)**：`#E5B535` (圣剑金)。**替换 WinUI 3 默认的系统主题色（SystemAccentColor）**。用于“启动按钮”、“Toggle 开关打开状态”、“左侧导航栏选中高亮指示条”以及“加载进度环 (Progress Ring)”。在暗色背景下形成强烈的视觉冲击。
* **辅助色 (Armor Silver)**：`#8A93A6` (铠甲银灰)。用于次要文本、面板边框、未激活的系统图标，增加金属质感。

### 3. Logo 与意象概念 (Visual Metaphors)
* **Logo**：建议设计为一把极简的几何形光剑自下而上穿透网络屏障（或无限符号 ∞），巧妙构成大写字母 **A** (Arturia) 的变体。
* **阿瓦隆 (Avalon)**：在设计“安全/设置”相关图标或空状态插画时，可融入多边形/六边形能量防御阵的设计语言。
* **微动效彩蛋**：主操作按钮（如“启动穿透”）在鼠标悬停 (Hover) 时，建议设计一道极快的斜向金属光泽（剑气）扫过按钮表面。

### 4. 核心交付页面需求
1. **应用图标 & Logo**（适配扁平化与微渐变质感）。
2. **启动页 (Splash Screen)**（深蓝背景加载框，包含 Logo 与 `Crafted by Arturia` 字样）。
3. **全局主界面 (Dashboard)**（包含左侧导航、核心高亮状态控制区、底部黑客风日志滚动面板）。
4. **穿透配置表单页**（排版干净整洁，输入框采用铠甲银灰描边）。
